API Architecture

Direct Integration.
No Off-System Shortcuts.

Kapitor APIs connect directly to core system modules — not to an abstraction layer. Every request touches the Unified Ledger. Every action has a compliance log, risk update, and audit record.

Webhooks & Event Streaming

Real-time notifications for transaction settlements, risk alerts, compliance flag updates, NAV changes, and instrument maturity alerts. Supports webhooks, streaming feeds, and batch exports. All events tied to ledger entries.

No Compliance Bypass Through Automation

API-triggered actions pass the same compliance gates as manual operations — KYC/KYB, AML screening, jurisdiction checks, investment limits, and risk thresholds. If compliance fails, the API returns a structured rejection.

Monitoring & Audit Exports

Institutional clients receive API usage logs, transaction audit trails, error tracking, rate limit metrics, and security event alerts. Audit exports are regulator-ready in structured format.

Versioning & Backward Compatibility

API versioning ensures backward compatibility across integrations. Changes are communicated via the developer portal changelog before deployment. No breaking changes without migration path.

Security Framework

Authenticated. Permissioned.
Every Request Logged.

◈
API Key + OAuth 2.0

Every integration uses an API key and secret model with OAuth 2.0 token authentication. Keys are scoped to entity, role, and permitted modules. No shared credentials across roles.

§
IP Whitelisting

Each API integration can define permitted IP ranges. Requests from non-whitelisted sources are rejected at the gateway — before authentication is attempted.

⬡
Request Signing

All API requests are signed using HMAC-SHA256. Signature verification runs before any processing occurs. Unsigned or invalidly signed requests return 401.

▣
Rate Limiting

Per-entity and per-role rate limits are enforced at the gateway. Burst limits apply to high-value operations. Rate metrics are visible in the institutional monitoring dashboard.

⊘
Encrypted Transport

All API traffic uses TLS 1.3 encryption in transit. Certificate pinning is available for enterprise integrations. No unencrypted API endpoints exist.

⇄
Audit Record Per Request

Every API request generates a timestamp, caller identity record, permission validation log, compliance entry, and audit record. All records are immutable and regulator-exportable.

◉
Multi-Approval Workflows

Organizations can enable multi-approval workflows for high-value or sensitive API actions. A single API call requiring two-person authorization will not execute until the second party confirms.

⬛
Transaction Size Caps

Per-role API-based transaction caps are set at the organizational level. The API cannot execute transactions beyond the defined cap without a policy change approved through governance.
