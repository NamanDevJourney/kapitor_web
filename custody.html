Custody Architecture

Three Layers. No Single
Point of Authorization.

Kapitor Custody operates under a structured multi-layer asset control model. Segregated accounts, multi-signature authorization, and cold-plus-hot infrastructure — each layer independently enforced.

Security Infrastructure

Security Is Architectural.
Not Reactive.

◈
Hardware-Based Key Management

Cryptographic keys are stored and operated within hardware security modules (HSMs). No software-level key exposure. All signing operations occur within secure hardware boundaries.

§
Network Segmentation

Custody infrastructure operates on isolated network segments, fully separated from operational and payment infrastructure. No shared network paths between cold storage and public-facing services.

⬡
Encrypted Storage

All custody records, transaction logs, and balance data are encrypted at rest and in transit. Encryption keys are rotated on governance-defined schedules.

▣
Anomaly Detection

Continuous behavioral anomaly detection monitors all custody access patterns. Deviations from established access baselines trigger temporary restrictions and escalation protocols.

⊘
Penetration Testing

Regular third-party penetration testing against custody infrastructure, key management systems, and access control layers. Findings remediated before next test cycle.

⇄
Continuous Monitoring

24/7 automated monitoring of custody balances, authorization requests, and access logs. Manual review workflows activated on anomaly detection. No unmanned custody windows.
